1️⃣ FCFS Scheduling

Concept: First Come First Serve – processes are executed in the order they arrive (non-preemptive).
Key Formula: TAT = CT − AT, WT = TAT − BT

2️⃣ SJF (Preemptive) Scheduling

Concept: Shortest Job First – CPU selects the process with the smallest remaining burst time. Preemption occurs when a new process with smaller BT arrives.
Drawback: May cause starvation for long jobs.

3️⃣ Priority Scheduling

Concept: CPU executes process with the highest priority (lowest numeric value). Non-preemptive version.
Issue: Starvation; can be reduced by aging.

4️⃣ Round Robin (Preemptive)

Concept: Each process gets equal fixed CPU time called Time Quantum (TQ). Preemptive scheduling based on cyclic order.
Note: If TQ → large ⇒ FCFS; if small ⇒ frequent context switches.

5️⃣ Memory Placement Strategies

Concept: Simulate memory allocation – First Fit, Best Fit, Worst Fit, Next Fit.
Goal: Efficiently assign processes to memory blocks while minimizing fragmentation.

6️⃣ Page Replacement Algorithms

Concept: Manage limited frames using FIFO, LRU, or Optimal page replacement.
Key Term: Page Fault – when a requested page is not in memory.
Note: Optimal is ideal, LRU is practical, FIFO is simple but can show Belady’s anomaly.

7️⃣ Producer–Consumer Synchronization

Concept: Classic synchronization problem using Semaphores/Mutex.
Goal: Ensure producer doesn’t overflow buffer and consumer doesn’t consume empty buffer.

8️⃣ Two Pass Assembler

Concept: Converts assembly language into machine code in two passes –
Pass I builds symbol/literal tables; Pass II generates final code using intermediate results.

9️⃣ Macro Processor

Concept: Expands macros before assembly; replaces macro calls with their definitions.
Goal: Code reuse, compact assembly programs.
